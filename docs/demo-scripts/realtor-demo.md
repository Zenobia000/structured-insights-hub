## Card 04｜現在怎麼解 + 為什麼還卡

| 欄位 | 內容 |
|---|---|
| **工具/方法名稱** | `LINE + Excel 看屋紀錄表 + 內部 CRM（信義房屋自己的系統）` |
| **第一版 workaround**（你先寫） | `每天早上看 Excel 標黃色（超過 3 天沒回的），優先傳 LINE。傳完不回的，週末再電話。` |

**AI 補的 5 個常見 workaround**（複製 prompt → AI 回 → 貼回來）：

```
1. 設定固定跟進節奏（24h / 72h / 1 週三波 SOP）
2. 用「房屋更新」當理由再次接觸（價格調整、新增照片、相似物件）
3. 透過共同好友/介紹人側面探詢
4. 寄實體 DM（手寫卡片 + 物件 brochure）
5. 邀請參加開放賞屋會，降低一對一壓力
```

**3 個不滿意（user 寫，必填）**：

| # | 不滿意這個 workaround 的原因 |
|---|---|
| 1 | `SOP 太機械化 — 客戶會感覺到被「催」，反而更不回。` |
| 2 | `用「房屋更新」當理由 — 同一物件講第二次還能 OK，第三次就明顯是話術。` |
| 3 | `寄實體 DM — 同事試過，30 封大概只有 1-2 封會回應，CP 值太低且慢。` |

## Card 06｜AI 證據蒐集（8 題）

頁面：複製這段 prompt → 丟到 Perplexity 或 ChatGPT (with web search) → 把回覆完整貼回來。

**Demo 用的 8 題完整答案**（假裝是 AI 回的，可直接貼）：

```
Q1（具體人群）：
中古屋首購族（30-38 歲，雙薪家庭，2500-3500 萬預算）、預售換屋族（40-50 歲，成屋需求）、投資客（45+，多戶經驗）。三類對「跟進」的接受度差異巨大。

Q2（場景與頻率）：
首購族每組看屋後 3-7 天「冷卻期」很明顯，這段時間 70% 的失聯發生於此。換屋族多在簽約前 1-2 週反覆比較。投資客則是價格沒到位就直接斷聯。

Q3（現有 workaround）：
LINE 訊息追蹤、CRM 自動排程提醒、共同好友介紹、實體 DM、品牌信任建立（持續 PO 成交實績）、被動等待客戶主動回頭。

Q4（不滿意的具體類別）：
- 機械化（SOP 訊息感覺像催收）
- 重複（同物件重複理由很尷尬）
- 慢（DM/介紹人需要 1-2 週）
- 沒區分人群（首購跟投資客用同套會兩邊都得罪）

Q5（公開證據連結）：
- Dcard 房仲版「客戶讀不回怎辦」相關發文約 40+ 篇（2025-2026）
- 591 房仲社群 FB「跟進節奏」討論串多次出現
- 商周 2025/11 報導「Z 世代買房不愛被 sales 追」
- 信義房屋內訓教材 2025 版專章「客戶冷卻期 SOP」

Q6（JTBD）：
房仲被「雇用」來在客戶決策的那 3-7 天「冷卻期」中，**幫他想清楚這間到底買不買，但不能讓他感覺被推銷**。

Q7（可能的假痛點）：
- 「客戶不回是因為房子貴」 → 其實只有 30% 是價格，70% 是還沒下決心
- 「強硬就會流失客戶」 → 區分客群後，投資客其實偏好強硬
- 「SOP 一套打天下」 → 三類客群需要不同節奏

Q8（訪談對象 5 種）：
1. 信義房屋同分店資深經紀（內部視角）
2. 跨品牌資深仲介（永慶/21世紀，外部對照）
3. 近 6 個月被「沒回」的真實客戶（買方視角，最珍貴）
4. 房仲教練/業務培訓老師（理論視角）
5. CRM 工具產品經理（工具視角）
```

**填回頁面的欄位**：

| AI tool | 選 `Perplexity`（demo 假裝用） |
|---|---|
| 為什麼選這個 tool | `要查 Dcard、591 社群這類即時討論，Perplexity 的 web search + 引用較準。` |
| no_solution_check_passed | ✓ 勾（AI 沒推銷產品/解法） |

| 欄位 | 內容 |

**3 個訪談對象**：

| # | Persona | 認識嗎 | 聯絡方式 / 怎麼找 | 預計時間 |
|---|---|---|---|---|
| 1 | `信義房屋同分店資深經紀（陳大哥）` | ✓ 認識 | `LINE: chenbro / 0987-114-258` | `2026-05-05 19:00 下班後熱炒` |
| 2 | `近半年被「沒回」的買方客戶（找王太太）` | ✓ 認識 | `LINE 主動傳「最近還在看嗎，純請教不推銷」` | `2026-05-10 週末咖啡 30 分` |
| 3 | `跨品牌資深房仲（永慶或 21 世紀）` | 我還不認識 | `去家附近永慶分店遞名片，請大哥介紹 2-3 位` | `2026-05-15 前約到` |

**訪題（3 條，照 §C8 訪談規則寫「現況問題」）**：

| # | 訪題 |
|---|---|
| 1 | `上一個你看完之後沒回我（或同事）的客戶，他最後是去買了別人的、還是放棄了？你怎麼知道的？` |
| 2 | `當你判斷一個客戶「該推一把」vs「該放空間」，是看什麼線索？最近一次你判斷錯的是哪個？` |
| 3 | `你跟同事最近有沒有討論「跟進節奏」？如果有，討論的 trigger 是什麼？` |

**勾選**：☑ 我看完了，知道訪談時不要做什麼（rules table）

**Stage 1 AI 模擬訪談**（demo 用，可貼）：

```
Q1 回答（陳大哥模擬）：
上週的張先生，我看他都不回，就放著。後來聽美玲說他在永慶簽了。問美玲怎麼知道的？她說那房子掛兩家，看買方資料對到了。所以我到現在也不知道我哪裡輸 — 是價格、是物件、還是我這人。

Q2 回答：
看「他主動問細節的數量」。如果他第二天主動問「停車位是平面還是機械」，我就知道有戲，可以推。如果只回「謝謝」，我就放空間。最近判斷錯的是李太太 — 她回得很積極，我推得很積極，結果她跟我說「我朋友昨天也帶我看一樣的，他比較專業」。

Q3 回答：
有，每週店會都會講。trigger 通常是「這週掉了 2 件以上單」就會討論。但討論完通常還是各做各的，因為每個客戶真的太不一樣。
```

**Stage 2 Audit findings**（AI 用 UX researcher 角度幫你 audit，可貼）：

```
[誘導性問題] Q2「該推一把 vs 該放空間」 — "推/放" 是你的詞，受訪者沒講過。
建議改：「你會在什麼狀況下主動聯絡客戶？什麼狀況下不會？」

[假設預設] Q3「最近有沒有討論跟進節奏」 — 預設了「跟進節奏」是個議題。
建議改：「你跟同事最近聊客戶聊最多的是什麼話題？」

[未挖掘的洞察] Q1 受訪者提到「兩家掛同案」這件事 — 這暗示「資訊透明度」可能是隱藏痛點，但你的 3 題沒有 follow-up 機會。
建議：保留 5 分鐘 free-flow 給受訪者自己延伸。

[強項] 三題都問「最近一次具體事件」— 符合 §C8.1「問現況不問意願」原則。
```

**Stage 3 Interview guide**（生成的訪綱 markdown）：

```markdown
# 房仲跟進節奏訪談 · v1
時長：30 分鐘 · 對象：陳大哥（信義同店）

## 暖場（3 min）
- 你最近一週帶看幾組？比平常多還少？

## 主軸（20 min）
1. 上一個你看完之後沒回的客戶，他最後是去買別人的還是放棄？你怎麼知道？（probe：你哪一刻發現他「冷下來」？）
2. 你會在什麼狀況下主動聯絡看完屋的客戶？什麼狀況下不會？（probe：可以舉一個最近的例子嗎？）
3. 你跟同事最近聊客戶聊最多的是什麼話題？（probe：是被 trigger 出來的，還是定期會講？）

## Free-flow（5 min）
- 還有什麼你覺得我應該問、但我剛剛沒問的？

## 結尾（2 min）
- 我整理完訪談重點會傳給你，可以嗎？
- 後續若我寫成文章/研究，會匿名你的內容，OK 嗎？
```
