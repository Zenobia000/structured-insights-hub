# PainMap Worksheet — Octalysis 動機設計總綱

> **配套文件**：`PRD.md`、`user_journey.md`、`stage1_to_stage2_handoff.md`
> **品牌守則**：`docs/web_design/global/painmap_brand_system.md`
> **核心立場**：白帽限定。永久禁用黑帽動機。違反禁令的設計**直接拒絕進入產品**。

---

## 0. 設計總則（讀任何細節之前先讀這段）

### 0.1 一句話總則

**Worksheet 唯一的動機來源是「把判斷力訓練好」這件事本身值得做**。任何外加的徽章、稀缺、競爭、收集、loot box，都會稀釋這個內在動機，必須拒絕。

### 0.2 三條鐵律

1. **白帽優於一切**：採用 Octalysis 八驅動力中的 #1、#2、#3、#5 四個白帽驅動力
2. **#4 Ownership 嚴格設限**：只能用於「資料主權」（你的本地資料、可匯出 / 刪除），禁止用於「收集慾」「IKEA 效應」誘導
3. **#6 Scarcity / #7 Unpredictability / #8 Loss Avoidance 永久禁用**：違反 PainMap brand「Anti-anxiety」原則

### 0.3 與 brand system 的對應關係

| `painmap_brand_system.md` 禁令 | Worksheet 動機設計回應 |
| :--- | :--- |
| 禁分數、星等、A-F 等級 | 不出現任何分數 UI（schema 內無 score 欄位）|
| 禁排行榜、徽章 | 不做使用者排名、不發徽章 |
| 禁遊戲化激勵 | 不用慶祝動畫、不用通關效果 |
| 禁焦慮數據分析 UI | 不用紅色警告、不用倒數計時 |
| Anti-anxiety 立場 | 永久禁用 #6/#7/#8 |
| 結構化、賦權、沉穩 | #1 意義 + #2 進度 + #3 創造力 + #5 社群（白帽四驅動力）|

## 1. 採用驅動力（白帽四驅）

### 1.1 #1 Epic Meaning & Calling — 史詩般的意義

#### 設計原則

讓使用者感受到「我在做的事比這 9 張卡片更大」。把判斷力訓練連結到「不被 AI 牽著走」「在 AI 時代保留自己的思考能力」這個更高敘事。

#### 在 Worksheet 的應用

| 應用點 | 實作 |
| :--- | :--- |
| 入口頁 hero 標語 | 「AI 時代最稀缺的不是工具、不是模型、不是資料——是判斷力」|
| 卡 7「自己先猜 vs AI」 | 介紹文字強調「這個練習在訓練你**不被 AI 順著走**」|
| 卡 9 真假判斷頁 | 文案「即便判斷為假，你也省下 3 個月走錯路的時間」|
| 痛點身份證匯出頁 | 標題「你的判斷力檔案」（不是「你的成績單」「你的 idea 清單」）|

#### Anti-pattern（要避免的偽 Epic Meaning）

- ❌ 「成為下一個改變世界的創業家」（虛假宏大）
- ❌ 「加入 10,000 個同樣有夢想的人」（蹭社群感而非真意義）
- ❌ 「不要讓你的 idea 被埋沒」（idea 焦慮，違反 brand 禁用詞）

#### 驗收標準

- [ ] 入口頁文案不出現「成功」「夢想」「改變世界」等空泛詞
- [ ] 每張卡片的引導文字至少 1 句連結到「判斷力訓練」這個更高目標
- [ ] 卡 9 假痛點判斷的文案是正向引導，不是失敗感

---

### 1.2 #2 Development & Accomplishment — 發展與成就感

#### 設計原則

讓使用者看見自己的進度，但**不用分數量化**，**不用徽章獎勵**。進度本身就是回饋。

#### 在 Worksheet 的應用

| 應用點 | 實作 |
| :--- | :--- |
| 9 卡 stepper 進度條 | 視覺化「我目前在第 N 卡」「已完成 X / 9」|
| 過關條件視覺化 | 每張卡片底部顯示「3 / 5 個欄位已完成」進度，但不用「分數」呈現 |
| 失敗回退路徑 | 退回上一卡時，已填寫資料保留（不歸零，避免懲罰感）|
| 痛點身份證 = 完成憑證 | 整張身份證以正式文件風格呈現，是「完成的證據」而非「分數的展示」|

#### 與 #2 設計但禁止越界的部分

- ✅ 進度條（顯示客觀完成度）
- ✅ 完成憑證（痛點身份證）
- ❌ 等級系統（如「青銅判斷者 / 白銀判斷者」）— 違反 brand 禁令
- ❌ 經驗值 XP — 違反 brand 禁令
- ❌ 解鎖機制（如「完成 3 份才解鎖第 4 種匯出格式」）— 製造稀缺感，違反 #6 禁令

#### Anti-pattern

- ❌ 「恭喜！你完成了第 5 張卡片！再 4 張就能拿到痛點達人徽章 🏆」
- ❌ 「你比 78% 的使用者快」（比較焦慮，違反 brand 禁令）
- ❌ 通關全螢幕慶祝動畫（遊戲化視覺）

#### 驗收標準

- [ ] Stepper 不使用百分比顯示「成就感」（純粹進度）
- [ ] 完成卡 9 時不使用慶祝動畫，只給平靜的「你的痛點身份證已就緒」
- [ ] 不出現任何等級 / 段位 / 徽章 UI 元素

---

### 1.3 #3 Empowerment of Creativity & Feedback — 創造力的賦權

#### 設計原則

讓使用者覺得「我在做選擇、我在創造判斷」，AI 是工具不是裁判。

#### 在 Worksheet 的應用

| 應用點 | 實作 |
| :--- | :--- |
| 卡 5 兩件事不能同時要 | 使用者用自己的話寫 A/B 兩端與被犧牲的原因（AI 協助釐清，不替你下判斷）|
| 卡 6 自選 AI 工具 | ChatGPT DR / Claude / Perplexity / Gemini 自選 + 給理由 |
| 卡 7 自己先猜 | UI 強制鎖住 AI 回覆，使用者必須先寫猜測 |
| 卡 8 自選訪談對象 | 從 AI 給的 5 種人裡挑 2 種，題目從題庫中挑 3 題（可改寫）|
| 卡 9 書面真假判斷 | AI 完全禁用，使用者自己寫 ≥ 100 字理由 |

#### 即時回饋設計

- 填欄位時即時顯示「這個欄位夠具體了 / 還可以再具體一點」（建議性而非評分性）
- AI 校對句型後，使用者可以選擇「用 AI 的版本 / 用我自己的版本」（不強制 AI 結果）
- 失敗回退時，告訴使用者**為什麼**，不只是說「你失敗了」

#### Anti-pattern

- ❌ AI 直接告訴使用者「這是真痛點 / 這是假痛點」（剝奪判斷權）
- ❌ AI 自動填卡 9 真假判斷（即便看似節省時間）
- ❌ 「AI 推薦你選矛盾 #2」標示為「最佳選擇」（暗示有正確答案）

#### 驗收標準

- [ ] 卡 9 真假判斷必須由使用者書面填寫（reason_100w 不可為 AI 生成）
- [ ] 卡 7 AI 回覆區必須在使用者寫完猜測後才解鎖
- [ ] AI 提案永遠以「參考」「提案」「常見可能」呈現，不用「推薦」「最佳」

---

### 1.4 #5 Social Influence & Relatedness — 社會影響與相關性

#### 設計原則

讓使用者感受到「我不是一個人在想這個問題」，但**不用排名 / 比較**製造焦慮。

#### 在 Worksheet 的應用

| 應用點 | 實作 |
| :--- | :--- |
| 痛點身份證可分享 | 匯出 Markdown / PDF，使用者可貼到部落格 / Notion |
| Pain Atlas 集體智慧入口（M2+）| 完成卡 9 後可選擇「貢獻到痛點圖譜」（匿名選項）|
| 範例案例（林老師家長 LINE）| 每張卡片有真實案例範例，讓使用者知道「別人這樣填」|
| 「我也遇到」共鳴按鈕（M2+）| 不是按讚 / 評分，僅表示共鳴不做排名 |

#### 與 #5 設計但禁止越界的部分

- ✅ 共鳴按鈕（無排名）
- ✅ 匿名分享（隱私優先）
- ✅ 集體智慧入口（self-opt-in）
- ❌ 排行榜（最痛 Top 10）— 違反 brand 禁令
- ❌ 比較表格（「你的痛點 vs 社群平均」）— 製造焦慮，違反 brand 禁令
- ❌ 朋友邀請獎勵（「邀請 3 個朋友送 Pro 版」）— 黑帽社群勒索

#### Anti-pattern

- ❌ 「Top 100 痛點排行榜」
- ❌ 「最熱門的痛點」（暗示熱門 = 好）
- ❌ 「你的判斷比 65% 使用者更深刻」（比較焦慮）

#### 驗收標準

- [ ] Pain Atlas 不出現排序的「最熱」「最高分」選項
- [ ] 範例案例不標榜「成功案例」「最佳實踐」
- [ ] 分享功能不附帶獎勵機制

## 2. 限制使用：#4 Ownership & Possession（嚴格界線）

### 2.1 唯一允許的應用：資料主權

| 允許 | 禁止 |
| :--- | :--- |
| ✅ 你的 PainCard 存在你的 LocalStorage，我們看不到 | ❌ 「收集所有 9 種卡片變成達人」(IKEA 收集慾) |
| ✅ 可隨時匯出 Markdown / JSON / PDF | ❌ 「你已經填了 X 張，再填 Y 張就能解鎖 Z」（鎖定機制）|
| ✅ 可隨時刪除（一鍵清除）| ❌ 「你的痛點身份證很特別，全世界只有你有」（虛假獨特感）|
| ✅ 不需要註冊帳號 | ❌ 「投入時間越多，價值越高」（IKEA 沉沒成本誘導）|

### 2.2 設計原則

- **資料主權 = 隱私 + 可攜性**：使用者擁有資料，可帶走可刪除
- **不引導收集**：即便使用者填了多份 PainCard，UI 上呈現為「歷史記錄」而非「收藏品」
- **不誘發 IKEA 效應**：投入再多時間，PainCard 也只是工具產出，不會被神聖化

### 2.3 Anti-pattern

- ❌ 「我的痛點收藏館」（收集慾）
- ❌ 「你已投入 12 小時製作這份痛點身份證」（沉沒成本暗示）
- ❌ 「升級 Pro 版解鎖無限痛點儲存」（人為稀缺）

### 2.4 驗收標準

- [ ] 多份 PainCard 列表 UI 不使用「收藏」「典藏」字眼
- [ ] 不顯示使用者已投入的時間
- [ ] 不對 PainCard 數量設上限（即便 LocalStorage 接近上限，也提示「匯出釋出空間」而非「升級擴容」）

## 3. 永久禁用（黑帽三驅）

### 3.1 #6 Scarcity & Impatience — 稀缺與不耐 — 永久禁用

#### 為什麼禁用

違反 PainMap brand 「Anti-anxiety」原則。Worksheet 是判斷力訓練工具，不應該製造時間壓力。

#### 禁用清單

| 禁用元素 | 為什麼禁 | 替代方案 |
| :--- | :--- | :--- |
| 倒數計時器（限時挑戰）| 製造焦慮 | 提示「平均 30–90 分鐘完成」（資訊性而非壓力性）|
| 限量名額（「今日只剩 100 個免費名額」）| 虛假稀缺 | 不設名額限制 |
| 限時優惠（「24 小時內升級 Pro 版」）| 強迫決策 | 不做時限優惠 |
| 解鎖機制（「完成 X 才能用 Y」）| 人為稀缺 | 全功能無條件提供 |
| 「機會稍縱即逝」文案 | 焦慮話術 | 「你隨時可以暫存退出，下次回來繼續」|
| Daily streak（連續使用獎勵）| 黑帽 streak 機制 | 不追蹤連續使用日 |
| 過期機制（「3 天內未完成則資料消失」）| 損失威脅 | LocalStorage 永久保留（除非使用者自刪）|

#### 驗收標準

- [ ] 全站無倒數計時器
- [ ] 全站無「限時」「限量」字眼
- [ ] 全站無 streak 計數
- [ ] LocalStorage 資料無自動過期機制

---

### 3.2 #7 Unpredictability & Curiosity — 不可預測性與好奇 — 永久禁用

#### 為什麼禁用

抽卡 / loot box 違反 Worksheet「結構化」核心。使用者期待**可預測的步驟**，不是賭博式驚喜。

#### 禁用清單

| 禁用元素 | 為什麼禁 | 替代方案 |
| :--- | :--- | :--- |
| 抽卡 / loot box 機制 | 賭博動機 | 不做隨機機制 |
| 隨機 prompt 推薦 | 結構化的反面 | 固定 7 段內建 prompt |
| 神秘成就（「完成隱藏條件解鎖」）| 不透明設計 | 所有 exit gate 明確列出 |
| 隨機抽訪談題（「AI 隨機出 3 題」）| 不可控 | 從卡 6 題庫中使用者主動挑 3 題 |
| Easter egg / 彩蛋 | 偏離核心價值 | 不做彩蛋 |
| A/B test 給不同使用者不同流程 | 違反一致性 | MVP 階段不做 A/B test（M2+ 嚴格規範下才做）|

#### 驗收標準

- [ ] 全站無隨機元素（除了 UUID 生成）
- [ ] 9 卡流程對所有使用者完全一致
- [ ] 每個 exit gate 條件公開可見

---

### 3.3 #8 Loss Avoidance — 損失規避 — 永久禁用

#### 為什麼禁用

製造焦慮，違反 brand「沉穩」立場。Worksheet 不威脅使用者「不做就會失去」。

#### 禁用清單

| 禁用元素 | 為什麼禁 | 替代方案 |
| :--- | :--- | :--- |
| 「資料即將過期」警告 | 損失威脅 | LocalStorage 永久保留 |
| 「不繼續就會失去進度」 | 焦慮話術 | 「你隨時可以暫存退出」|
| 「Streak 即將斷裂」 | 黑帽 streak | 不追蹤 streak |
| 「你已投入 X 小時，現在放棄太可惜」 | 沉沒成本誘導 | 「換題目重新填，這次更熟練」|
| 「假痛點 = 浪費」文案 | 失敗感 | 「假痛點 = 你省下的時間」|
| 「不分享給朋友 = 浪費這份洞察」 | 社群勒索 | 不誘導分享 |

#### 驗收標準

- [ ] 全站無「失去」「過期」「浪費」「可惜」等損失話術
- [ ] 卡 9 假痛點判斷文案為正向引導
- [ ] 不對中斷使用者發送提醒通知（無 push notification）

## 4. 9 卡片動機驅動力對照表

| 卡 | 主驅動力 | 副驅動力 | 設計重點 |
| :-- | :--- | :--- | :--- |
| 1 抱怨原句 | #1 Epic Meaning | #4 Ownership（資料主權）| 「這是你聽到的真實聲音」強調原句不可被替代 |
| 2 三個有名字的人 | #1 Epic Meaning | #2 Accomplishment | 「這是判斷力的第一個物理量」連結到更高目標 |
| 3 卡關公式 | #3 Empowerment | #2 Accomplishment | 使用者選擇「用 AI / 用自己」版本，給控制權 |
| 4 現在怎麼解 | #5 Social Influence | #3 Empowerment | 「拿這 5 個 workaround 去問主人翁」促真人連結 |
| 5 兩件事不能同時要 | #3 Empowerment | #1 Epic Meaning | 用主人翁的話寫 A/B 兩端與被犧牲的原因，AI 只是參考 |
| 6 AI 證據蒐集 | #2 Accomplishment | #3 Empowerment | 自選 AI 工具 + 8 題結構化證據 |
| 7 自己先猜 + 讀 AI | #3 Empowerment | #1 Epic Meaning | 「先猜後看」是判斷力訓練的核心 |
| 8 真人訪談規劃 | #5 Social Influence | #1 Epic Meaning | 「AI 不能取代真人訪談」社群連結 |
| 9 真假判斷 | #2 Accomplishment | #1 Epic Meaning | 完成憑證 = 痛點身份證；意義感 = 判斷力 |
| 10 痛點身份證 | #5 Social Influence | #4 Ownership（資料主權）| 可分享 + 可匯出 |

### 設計原則

- 每張卡片**最多** 2 個驅動力（避免過度設計）
- 卡 1、5、7、9 是「核心判斷力訓練」卡，主驅動力以 #1 / #3 為主
- 卡 6 是「AI 介入最多」卡，副驅動力 #3 確保使用者不被 AI 主導
- 卡 10 是「對外」卡，社群與資料主權並重

## 5. 設計反模式（Anti-pattern）— PR Review 必檢清單

### 5.1 偷渡黑帽動機的常見手法

| Anti-pattern | 表現 | 為什麼是黑帽偷渡 |
| :--- | :--- | :--- |
| 「再 1 卡就完成！」過度推送 | 文案過度強調進度 | 隱含 #6 稀缺感 |
| 「您是第 1024 個完成的使用者」 | 加入排名 | 違反 brand 禁令 |
| 「7 天未填完的會自動封存」 | 時間壓力 | 違反 #6 / #8 禁令 |
| 「升級 Pro 版解鎖無限 PainCard」 | 人為限制 | 違反 #4 Ownership 設限 + #6 |
| 「AI 模擬訪談每日只能用 3 次」 | 限量機制 | 違反 #6 |
| 「分享身份證給 3 個朋友解鎖匯出 PDF」 | 社群勒索 | 違反 #5 設計原則 |
| 「歡迎回來！你的 streak 是 3 天」 | streak 機制 | 違反 #8 |
| 「猜猜你下一步該怎麼做？」隨機 tooltip | 不可預測 | 違反 #7 |
| 卡 9 顯示「你的分數是 24 / 25！」全螢幕慶祝 | 過度成就 | 違反 brand 禁令 + #2 過度設計 |
| 「90% 的使用者在這一卡放棄，你能撐住嗎？」 | 競爭挑釁 | 違反 #5 設計原則 |

### 5.2 偽白帽（看似白帽其實是黑帽）

| 偽白帽 | 為什麼是黑帽 |
| :--- | :--- |
| 「Epic Meaning：成為下一個改變世界的人」 | 虛假宏大，違反 brand「不空泛」原則 |
| 「Accomplishment：解鎖『判斷力大師』成就」 | 徽章機制，違反 brand 禁令 |
| 「Empowerment：你的選擇將決定 100 萬人的未來」 | 誇大賦權，製造心理負擔 |
| 「Social：你的痛點身份證已被 50 人讚」 | 量化社群認可，違反 #5 原則 |

### 5.3 灰區判斷準則

當不確定一個設計是白帽還是黑帽時，問三個問題：

1. **這個設計如果消失，使用者會不舒服嗎？**
   - 是 → 可能是黑帽（製造依賴）
   - 否 → 可能是白帽
2. **這個設計是讓使用者更清楚自己在做什麼，還是更想留下來？**
   - 更清楚 → 白帽
   - 更想留下來（不知為何留）→ 黑帽
3. **這個設計能不能用「沉穩」「結構化」「賦權」三個 brand 形容詞描述？**
   - 三個都能 → 白帽
   - 只能用一個或都不能 → 重新檢視

## 6. 與其他設計禁令的對應

### 6.1 與 PainMap brand 禁令的對應

| brand 禁令 | 對應的 Octalysis 黑帽動機 | Worksheet 處理 |
| :--- | :--- | :--- |
| 禁分數、星等 | #2 過度設計（成就量化）| schema 內無 score 欄位 |
| 禁排行榜 | #5 競爭機制 | 完全不做排名 |
| 禁徽章 | #4 收集慾 + #2 過度成就 | 不發徽章 |
| 禁 FOMO | #6 稀缺 + #8 損失規避 | 全站禁用 FOMO 文案 |
| 禁焦慮 UI | #6 / #8 | 全站禁用紅色警告（除系統錯誤）|

### 6.2 與 painpoint_beginner_worksheet.md（紙本）的對應

紙本 worksheet 本身就是白帽設計（沒有任何遊戲化元素）。網頁版的責任是**不破壞**紙本的純淨度，而不是「加入更多動機」。

設計原則：

- 紙本沒做的事情，網頁版**不主動加**（除非有清楚的白帽理由）
- 紙本做的事情，網頁版**完全保留**
- 網頁版多出來的功能（自動儲存、prompt 一鍵複製、匯出格式）必須通過白帽檢核

## 8. 設計者備忘（給未來貢獻者）

### 8.1 為什麼這份文件這麼嚴格

因為**動機設計很容易偷渡**。一個小小的「再做 1 卡就完成」推播訊息，可能就把整個工具從白帽推向黑帽。Worksheet 的價值在於它**不會讓使用者上癮**，使用者用完即走是好事。

### 8.2 為什麼不採用 #4 Ownership 完整版

完整 #4 包含「IKEA 效應」「收集慾」「客製化所有權」等子驅動力，這些子驅動力很容易越界成黑帽（如「你已經投入 100 小時，不能放棄」沉沒成本陷阱）。我們選擇只用「資料主權」這個最安全的子應用。

### 8.3 為什麼不做更多 #5 Social

Worksheet 是個人工具，不是社群產品。過度社群化會稀釋「個人判斷力訓練」這個核心。Pain Atlas 是 M2+ 的進階版，並非 MVP 必要。

### 8.4 為什麼禁用 #6/#7/#8 比禁分數更重要

分數是一種顯性的黑帽（容易識別）。#6/#7/#8 是隱性的黑帽（容易偷渡）。一旦這三個進入產品，使用者會在無意識中被操縱，失去白帽工具的尊嚴。

### 8.5 給未來新增功能的人的話

新增任何功能前，先回答：

1. 這個功能是讓使用者**更清楚**還是**更上癮**？
2. 這個功能能不能用「沉穩、結構化、賦權」三個詞描述？
3. 這個功能消失了，使用者會更焦慮還是更平靜？

如果答案不清楚 → **不要做**。Worksheet 的價值不在功能多，而在純淨。

---

> **最後一句**：白帽動機的最高境界，是讓使用者**不知道自己被動機驅動**。他們只知道：「這個工具尊重我，我願意花時間用它」。這就夠了。
