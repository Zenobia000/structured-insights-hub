# Anti-Gamification Guardrails — 黑帽禁令清單

> **真相源**：
> - `docs/web_design/global/painmap_brand_system.md` 第 24-30 行「絕對禁令」 + 第 359-374 行「ANTI-PATTERN REGISTRY」
> - `docs/superpowers/specs/2026-05-01-painmap-worksheet-design.md` 第 134-139 行「永久禁用（黑帽 3 個）」
> - `.claude/skills/sunnydata-pain-thinking/SKILL.md` 第 220-231 行 Anti-patterns
>
> **本檔角色**：永久禁用的 3 個黑帽 Octalysis 驅動力（#6 Scarcity / #7 Unpredictability / #8 Loss Avoidance）的完整禁令清單，包含：偽裝設計、PR review 必抓關鍵字、灰色地帶判定。
>
> **設計鐵律**：本清單**永久有效**。任何引入這 3 個驅動力的設計，無論包裝如何巧妙，都不能進入產品。

---

## 0. 為什麼要做這份清單？

PainMap 品牌核心是「**結構化、賦權、沉穩 — 拒絕焦慮、拒絕遊戲化激勵**」。

但黑帽 Octalysis 驅動力非常容易偷渡：
- 「14 天免費」聽起來很佛心，其實是 #6 稀缺
- 「每日簽到連續獎勵」聽起來像養成，其實是 #8 損失恐懼
- 「神秘禮物 / 抽 AI 工具」聽起來像驚喜，其實是 #7 不可預測

這份清單的目的：**讓 PR reviewer 一眼識破偽裝**。

## 1. 永久禁用驅動力 — 完整清單

### 1.1 #6 Scarcity & Impatience（稀缺與不耐）

**定義**：透過稀缺感（數量有限 / 時間有限）製造急迫感，驅使使用者立即行動。

**禁用原因**：

> 違反 PainMap brand「Anti-anxiety」原則（spec 第 137 行）。
> PainMap 的核心是「沉穩、容許不確定、給使用者思考空間」。稀缺感強迫使用者放棄思考、立即行動，與品牌目標完全相反。

**對應 PainMap brand 禁令**（painmap_brand_system.md 第 25-29 行）：
- 禁：「焦慮催促」「遊戲化激勵」（第 78 行）
- 禁：「倒數計時器 — 人為緊迫感」（第 372 行）
- 禁：「FOMO 話術」（spec 第 35 行）

#### 容易被偷渡的偽裝設計

| 偽裝形式 | 為何是 #6 | 識別關鍵 |
| :--- | :--- | :--- |
| 「限時免費 14 天」 | 「14 天」= 稀缺時間；「限時」= 急迫感 | 任何「N 天 / N 小時」+「限」字 |
| 「前 100 名免費」 | 數量有限 | 「前 N 名」「限量 X 份」 |
| 「今日截止」「優惠倒數」 | 時間壓力 | 「截止」「倒數」「剩餘」 |
| 「快速通道：3 分鐘填完」 | 速度競爭催促 | 「快速 / 加速」搭配時間數字 |
| 「最後 5 個席次」 | 數量稀缺 | 「最後 N 個」「剩 N 名」 |
| 「現在加入解鎖進階」 | 「現在」= 急迫；「解鎖」= 暗示稀缺 | 「現在」「立即」+「解鎖」 |
| 「VIP 專屬」 | 身份稀缺 | 「VIP / 專屬 / 獨享」 |
| 「神秘高手挑戰」 | 神秘感 + 進階感 | 「神秘」「進階限定」 |

#### PR review 必抓關鍵字

```
限時、限量、限定、僅限、最後、剩餘、剩下、截止、倒數、倒計時、
僅有 N 個、僅 N 名、僅今日、24 小時內、本週限定、月內截止、
立即、馬上、現在、現在不、再不、機不可失、錯過、過時、
快速通道、加速、優先權、VIP、專屬、獨享、首發、
```

#### PR review 必抓 UI pattern

- 倒數計時器（Countdown timer）
- 進度條附帶截止時間
- 「N 人正在搶 / 剛剛有 N 人購買」即時通知
- 「Hurry up」「Don't miss」型微文案
- 紅色 / 橘色高亮的「LIMITED」「LAST CHANCE」標籤
- 動畫倒數（數字跳動）

#### Worksheet 的正確對照

worksheet 第 9 行：「準備時間：第一次 90 分鐘；熟練後 30 分鐘。」

✅ 為什麼這不算 #6：
- 給的是「事實預估」（這是花多久），不是壓力
- 沒有截止時間（不是「90 分鐘內必須完成」）
- 不會在使用者超時時懲罰 / 警告
- 無倒數計時器

### 1.2 #7 Unpredictability & Curiosity（不可預測與好奇）

**定義**：透過隨機獎勵（loot box、抽卡、神秘禮物）操弄好奇心與多巴胺，驅使使用者重複行動。

**禁用原因**：

> 違反 PainMap brand「結構化」核心（spec 第 138 行原文：「抽卡 / loot box 違反『結構化』核心」）。
> PainMap 的核心是「把混亂的痛點轉化為清晰的問題格式」。任何隨機 / 抽獎 / 神秘獎勵，都會讓使用者把焦點從「結構化思考」轉移到「下一個會抽到什麼」。

**對應 PainMap brand 禁令**：
- 禁：「成功率預測、可行性百分比」（第 26 行）
- 禁：「遊戲化徽章」（第 28 行）
- 禁：「比較表格」（第 373 行）

#### 容易被偷渡的偽裝設計

| 偽裝形式 | 為何是 #7 | 識別關鍵 |
| :--- | :--- | :--- |
| 「神秘 AI 助手陪你練習」 | 「神秘」= 不可預測 | 任何「神秘 / 驚喜 / 抽」字眼 |
| 「每日抽 1 個免費 prompt 範本」 | 抽獎機制 | 「抽」「轉盤」「Loot」 |
| 「隨機推薦痛點 idea」 | 隨機推薦本質 | 「隨機」「為你推薦」附驚喜元素 |
| 「打開禮物盒看你今天的洞察」 | 禮物盒 = loot box | 「禮物盒」「神秘盒」「百寶箱」 |
| 「彩蛋：填到第 7 張卡有特殊內容」 | 彩蛋 = 不可預測獎勵 | 「彩蛋」「Easter Egg」「驚喜」 |
| 「每天上線都有不同的提示」 | 變動性 = 操弄好奇 | 「不同 / 變動 / 隨機」+ 訪問頻率 |
| 「成功率 78%」 | 假精確 = 好奇驅動 | 任何「N%」用於品質評估 |
| 「神級判斷」「大師級洞察」 | 等級神秘感 | 「神級 / 大師 / 傳說」 |

#### PR review 必抓關鍵字

```
神秘、神級、大師、傳說、隱藏、彩蛋、驚喜、禮物盒、百寶箱、
抽、抽獎、轉盤、Loot、Lucky、Random、隨機、為你推薦、
今日特選、每日新發現、不同、變動、變化、看看你今天會抽到什麼、
解鎖內容、特殊內容、限定彩蛋、
N% 機率、N% 成功率、N% 可能、
```

#### PR review 必抓 UI pattern

- 抽獎轉盤動畫
- 卡包打開動畫（loot box opening）
- 「？？？」遮蔽內容直到完成某動作
- 機率顯示（「30% 解鎖紫色卡片」）
- 隨機跳轉首頁內容
- 「你今天的幸運痛點」式個性化推薦

#### Worksheet 的正確對照

worksheet 整體完全不依賴 #7。9 張卡片、6 個矛盾、5 個維度，全部**結構化、可預測、透明**。

✅ 為什麼這對使用者更好：
- 使用者知道每一步會發生什麼
- 沒有「等等不知道會跑出什麼」的焦慮
- 訓練判斷力靠的是穩定流程，不是好奇驅動
- 即使重做 10 次，流程仍一樣（不是賭博）

### 1.3 #8 Loss & Avoidance（損失與逃避）

**定義**：透過「失去恐懼」（streak 中斷、過期失效、進度歸零）驅使使用者持續使用，避免損失。

**禁用原因**：

> 違反 PainMap brand「Anti-anxiety」原則（spec 第 138 行原文：「streak / 過期 = 製造焦慮」）。
> PainMap 不希望使用者「不得不繼續」。判斷力訓練是自願的、可中斷的、不會因為休息而失去資格。

**對應 PainMap brand 禁令**：
- 禁：「任何引發焦慮的數據分析 UI」（第 29 行）
- 禁：「FOMO 話術」（spec 第 35 行）
- 禁：「比較表格（你的 vs 他人的）」（第 373 行）

#### 容易被偷渡的偽裝設計

| 偽裝形式 | 為何是 #8 | 識別關鍵 |
| :--- | :--- | :--- |
| 「每日簽到連續獎勵」 | 中斷 = 損失 streak | 「連續」「streak」「不間斷」 |
| 「7 天不上線進度歸零」 | 過期失效 | 「歸零」「失效」「重新計算」 |
| 「3 天內未完成將消失」 | 強制完成 | 「未完成將」「自動刪除」 |
| 「連續 N 天填卡解鎖徽章」 | 中斷即失去 | 「連續 N 天」+「解鎖」 |
| 「你已連續 5 天沒上線了」 | 喚回式焦慮 | 「已連續 N 天沒...」 |
| 「再不繼續就要從頭開始」 | 進度威脅 | 「從頭」「重來」威脅 |
| 「你的 progress 將在 N 天後過期」 | 進度過期 | 「過期」「失效」「outdated」 |
| 「離開前要把卡片完成嗎？」（強烈挽留訊息）| 失去焦慮 | 過度挽留 |
| 「你比上週退步了 N 名」 | 排名損失 | 退步比較 |

#### PR review 必抓關鍵字

```
連續、streak、不間斷、累積、保持紀錄、保持連勝、別斷了、
過期、失效、歸零、重來、從頭、重新計算、清空、
快過期、即將過期、再不繼續、還剩 N 天、
你已 N 天沒上線、你已 N 天未填卡、你的進度需要刷新、
失去資格、失去等級、降級、退步、後退、
比上週退、比上次少、本週你只完成、（鼓勵性比較皆禁）
```

#### PR review 必抓 UI pattern

- 連續登入記錄圖（streak calendar）
- 「再 N 天就連續 30 天！」進度條
- Email / Push 「你已 N 天沒上線」
- 「進度即將過期」紅色警告
- 強制挽留 modal：「真的要離開嗎？你的進度將...」
- 「最後一次活動：N 天前」（搭配警告色）
- 比較圖表：「本週 vs 上週」「你 vs 平均」

#### Worksheet 的正確對照

worksheet 設計上完全不依賴 #8。

✅ 為什麼這對使用者更好：
- 卡片可以任意暫停、回頭修改、放棄重來，沒有懲罰
- 沒有「N 天不填會清空」（資料永遠在）
- 沒有「連續 N 天」獎勵（不需要每天回來）
- 卡 9 過關條件是書面理由，不是「按時完成」
- 失敗回退（exit_gates_matrix.md）的訊息都是賦權式：「先存草稿吧」、「5 分鐘可以做完」，不是「再不做就...」

## 3. 灰色地帶處理原則

某些設計表面上像白帽，實際上會偷渡黑帽。這節列出最常見的灰色地帶判定。

### 3.1 Progress bar：是 #2 還是 #8？

**判定規則**：

| 設計 | 判定 |
| :--- | :--- |
| 顯示「3 / 9 卡片完成」 | ✅ #2 白帽（事實） |
| 進度條附帶倒數計時器 | ❌ #6 + #8（時間壓力） |
| 進度條顯示「再 X 天會過期」 | ❌ #8（過期恐懼） |
| 進度條顯示「比上週進度多 N」 | ❌ #5 + #8（比較焦慮） |
| 進度條中斷會 reset | ❌ #8（中斷懲罰） |
| 進度條可中斷、可儲存草稿 | ✅ #2（無懲罰） |

**通用原則**：填寫進度視覺化 OK，但不可加倒數計時器、不可中斷懲罰。

### 3.2 視覺反饋：是 #3 還是 #7？

| 設計 | 判定 |
| :--- | :--- |
| 過關時 checkmark 動畫 | ✅ #3 賦權回饋 |
| 過關時抽卡牌動畫 | ❌ #7 抽獎 |
| 過關時顯示「下一張卡片」 | ✅ #3 |
| 過關時隨機獎勵動畫 | ❌ #7 |
| 過關時顯示「你已完成 N / 9」 | ✅ #2 事實 |

### 3.3 完成激勵：是 #2 還是 #2 偷渡？

| 設計 | 判定 |
| :--- | :--- |
| 顯示「痛點身份證已完成」 | ✅ #2 完成憑證 |
| 顯示「你的等級：Lv. 1 痛點分析師」 | ❌ #2 偷渡（等級遊戲化） |
| 顯示「你已完成 5 個痛點身份證」 | ✅ #2 事實 |
| 顯示「金牌痛點獵人徽章解鎖」 | ❌ #2 偷渡（徽章） |
| 顯示「下一個目標：填第 6 張卡片」 | ✅ #2 引導下一步 |
| 顯示「再填 3 張就可升級！」 | ❌ #2 + #8 偷渡（升級話術） |

### 3.4 社群展示：是 #5 還是 #5 偷渡？

| 設計 | 判定 |
| :--- | :--- |
| 「我也遇到」按鈕（共鳴） | ✅ #5 白帽 |
| 「按讚 / 評分」 | ❌ #5 偷渡（排名） |
| Pain Atlas「最新」「最多共鳴」排序 | ✅ #5 白帽 |
| Pain Atlas「最熱門」「最高分」排序 | ❌ #5 偷渡（排名） |
| 顯示貢獻者匿名身分 | ✅ #5 + #4 資料主權 |
| 顯示「Top 10 Contributor」榜單 | ❌ #5 偷渡（排行榜） |
| 「分享你的痛點身份證」按鈕 | ✅ #5 白帽 |
| 「分享後解鎖更多功能」 | ❌ #5 偷渡（強制分享） |

### 3.5 提醒通知：是引導還是 #8 焦慮？

| 設計 | 判定 |
| :--- | :--- |
| 「你有 1 張卡片未完成，要繼續嗎？」 | ✅ 中性引導 |
| 「你已 7 天沒填卡了，回來吧！」 | ❌ #8 喚回焦慮 |
| 「Pain Atlas 有新增 3 個類似痛點」 | ✅ #5 白帽通知 |
| 「再不上線你的 streak 會中斷！」 | ❌ #8 streak 焦慮 |
| 「你的草稿已存 30 天，可以匯出或刪除」 | ✅ 資料主權告知 |

## 4. 違規處理流程

### 4.1 誰可以核准例外？

**原則**：本清單**不接受例外**。任何違反 §1.1 / §1.2 / §1.3 的設計，無論誰提出，都不能進入產品。

**唯一可討論的情況**：如果有新的學術研究 / 使用者研究證明「某個被列為黑帽的設計，在 PainMap 情境下對使用者真的有正面影響」，需要：

1. 提出書面證據（論文、使用者測試報告）
2. PainMap brand owner（Sunny）+ 至少 2 位審查人簽核
3. 修訂本清單 §1（記錄為例外條款），不私下放行

### 4.2 PR review 違規處理

當 reviewer 發現違規：

```
1. 標記 PR 為「Needs revision」
2. 在 PR comment 引用本檔對應段落（如 §1.1.PR-keywords）
3. 提供白帽替代方案（從 octalysis_white_hat_principles.md §文案模板找）
4. 不直接 reject，給 PR 作者修改機會
```

### 4.3 已上線發現違規

```
1. 立即建立 hotfix branch
2. 移除違規元素（不替換為「弱化版黑帽」）
3. 補上對應白帽設計（如有需要）
4. 在 CHANGELOG 紀錄移除原因
```

### 5.1 程式碼層

PR CI 加入 lint rule，掃描檔案中是否含有：

```js
const FORBIDDEN_PATTERNS = {

// 在 .tsx / .ts / .md 文件中找出 → 自動 fail PR
```

QA stage 額外檢查：

## 6. 為什麼黑帽驅動力對 PainMap 特別致命？

PainMap 訓練的是**判斷力**。判斷力需要的條件：

| 判斷力的條件 | 黑帽驅動力的破壞 |
| :--- | :--- |
| 思考時間 | #6 稀缺強迫立即行動 |
| 穩定流程 | #7 不可預測引入隨機性 |
| 自願參與 | #8 損失恐懼變成被綁架 |
| 結構化思考 | #6 / #7 / #8 都讓使用者把心力放在獎勵 / 截止 / 不要中斷 |
| 拒絕假答案 | 黑帽會獎勵「快速給答案」，不獎勵「我說我不確定」 |

PainMap brand 的「沉穩、賦權、結構化」三個關鍵字，每一個都與黑帽驅動力直接衝突。

## 8. 常見問題（FAQ）

### Q1：「14 天免費試用」這種商業模式上常見的設計也禁嗎？

A：商業頁面（landing / pricing）不在本檔範圍內。本檔守的是 worksheet 9 卡 + 痛點身份證流程。如果未來開放付費版，pricing 頁面的「14 天免費」需要另立規範討論。但即使如此，「免費期間倒數紅字提示」這類 UI 仍應避免。

### Q2：「這張卡片預估 5 分鐘」這算 #6 嗎？

A：不算。給「事實預估時間」是引導性資訊，不是壓力。判定差異：是否有截止懲罰？沒有 → 安全。

### Q3：「連續完成 5 張卡片解鎖整體預覽」算 #8 嗎？

A：算。「連續完成 N」+「解鎖」就是 streak 偽裝。改寫為「完成 5 張卡片後可預覽整體」即可（去掉「連續」）。

### Q4：「Pain Atlas 顯示『熱門痛點』」算違規嗎？

A：算。`painmap_brand_system.md` Pattern 4 第 354 行明文：「無排序選項中不包含『最熱門』『最高分』 — 僅提供『最新』『最多共鳴』『你可能相關』」。改用「最多共鳴」即可。

### Q5：教學模式可以顯示 0-25 score 嗎？

A：可以，但僅限教學模式內部反思鏡子，且必須附上 disclaimer：「分數只是鏡子，不是答案。24 分的痛點仍可能是假；14 分的抱怨仍可能是真。」（pain_card_schema.md R4.1、SKILL.md Teaching mode 段落）。生產模式禁止輸出 score。

### Q6：「你已完成第 3 張卡片，下一張是 ___」算激勵嗎？

A：✅ 安全。這是 #2 白帽（Development & Accomplishment）的正確用法 — 事實 + 引導下一步，不評價、不比較、不威脅。
