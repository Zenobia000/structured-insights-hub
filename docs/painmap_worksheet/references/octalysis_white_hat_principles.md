## 1. 驅動力 #1 ｜ Epic Meaning（史詩般的意義）

### 1.1 應用範圍

PainMap Worksheet 的 #1 體現於品牌敘事：「**判斷力訓練 / 不被 AI 牽著走**」。

我們希望使用者填完 9 張卡片後，認知到：

- 自己練的不是「會用 AI」，是「在 AI 時代仍能下判斷」
- 這份卡片不是「找答案的工具」，是「**判斷力訓練器**」（worksheet 第 39 行）
- 真假痛點的判斷，永遠是人的職責

### 1.2 具體實作

| 元素 | 設計 |
| :--- | :--- |
| Landing Page Hero | Headline：「AI 時代最稀缺的不是工具，是判斷力。」（worksheet 第 745 行原文） |
| Sub-headline | 「9 張卡片，訓練你從一句抱怨判斷它是不是真痛點。」 |
| 開場 30 秒短文 | 直接引用 worksheet 第 14-44 行的「30 秒搞懂這份東西」 |
| 完成卡 9 後的訊息 | 「你完成了一次完整的判斷力訓練。這張身份證代表你已經能從一句抱怨拆出真假。」 |
| 痛點身份證封面文案 | 「不是給 AI 看的紀錄，是給未來的你看的判斷」 |

### 1.3 文案模板（10 個可直接套用）

```
1. 「AI 時代最稀缺的不是工具、不是模型、不是資料 — 是判斷力。」
2. 「這份不是教你做產品，是訓練你判斷一個痛點是真是假。」
3. 「你不需要懂創新理論。你只需要會聽真人怎麼說、怎麼卡。」
4. 「AI 會幫你整理、提案、查證據、出題；但『真的嗎、值得嗎』這兩題，永遠是你來判。」
5. 「這張身份證裡沒有『錢』。判斷力訓練不需要先收錢、先架網站、先做 App。」
6. 「判定真痛點 → 接著去做訪談。判定假痛點 → 開心地丟掉，去下一張卡片。」
7. 「9 張卡片，訓練 1 件事：判斷一個痛點是真是假。」
8. 「真痛點是用真人說出來的話拼出來的，不是腦袋裡想出來的。」
9. 「我們相信：你的判斷比 AI 的答案更值錢。」
10. 「判斷力是練出來的。9 張卡片是你的小器材。」
```

### 1.4 正確用法 vs 偷渡黑帽 對照

| ✅ 正確 | ❌ 偷渡黑帽 |
| :--- | :--- |
| 「你正在訓練自己的判斷力」 | 「成為判斷力大師！」（自我膨脹話術） |
| 「AI 時代最稀缺的是判斷力」 | 「再不練判斷力，你會被 AI 取代！」（FOMO） |
| 「這張身份證代表你做完一次完整訓練」 | 「比 80% 的人更懂判斷痛點」（社會比較） |
| 「判斷一個痛點是真是假」 | 「成為頂尖痛點獵人」（角色扮演幻想） |

## 2. 驅動力 #2 ｜ Development & Accomplishment（發展與成就）

### 2.1 應用範圍

把「9 步判斷流程」做成可視化的進度系統，讓使用者每填完一張卡都看到「自己往前走了一步」。

> ⚠️ 嚴格禁止：等級、分數、徽章、星等、A-F 評分。這些是 Development 的反模式。

### 2.2 具體實作

#### 2.2.1 9 步進度視覺化（Progress Stepper）

```
卡 1   卡 2   卡 3   卡 4   卡 5   卡 6   卡 7   卡 8   卡 9
●─────●─────●─────○─────○─────○─────○─────○─────○
已完成 已完成 進行中  待開始 待開始 待開始 待開始 待開始 待開始
```

設計守則：

- 視覺：填滿 vs 空圈（不可用百分比 / 燈號 / 等級）
- 顏色：完成用 `--color-verified: #2D9D78`（驗證綠）；進行中用 `--color-secondary: #2D7D8A`（Teal）
- 不顯示「你超越了 N% 的人」
- 不顯示倒數計時 / 預估時間（怕觸發 #6 焦慮）
- 顯示「你已完成 X / 9 張卡片」（事實，不評價）

#### 2.2.2 痛點身份證 = 完成憑證（最終 Output）

worksheet 第 565 行的「痛點身份證」就是 Accomplishment 的最終呈現：

```
═══════════════════════════════════
  痛點身份證
═══════════════════════════════════
主人翁：林老師
場景：...
我的判斷：☑ 真痛點
建立日期：2026-04-15
═══════════════════════════════════
```

設計守則：

- 不出現「Lv. 1 痛點獵人」「金牌痛點挑戰者」這類等級
- 不出現任何「N / 25」「N / 100」型總分（schema 內無 score 欄位，匯出格式也不衍生分數）
- 可匯出為 Markdown / JSON / PDF（給使用者擁有感）

#### 2.2.3 過關條件視覺化

每一張卡都有 Exit Gate（見 `exit_gates_matrix.md`）。設計上：

- 過關條件以 checklist 呈現：
  ```
  ☑ 寫的是原句不是解釋
  ☑ 至少有 1 個有名字的真人
  ```
- 條件全勾 → 「下一步」按鈕變成 Active（從 disabled 變 enabled）
- 不勾選用「✅」打勾標誌，不用分數 / 進度條

### 2.3 文案模板

```
1. 「你已完成 3 張卡片。下一張：找出他現在怎麼解。」
2. 「這張卡片過關了。下一步：把卡關拆成卡關公式。」
3. 「9 張卡片裡，你已經寫到第 6 張。剩下 3 張就完成判斷。」
4. 「你的痛點身份證已準備好。可以匯出 / 列印 / 分享。」
5. 「第一次填到這裡通常需要 90 分鐘。你目前已經花了 _ 分鐘。」（事實 metadata，不施壓）
6. 「過關條件全部達成。下一張卡片解鎖。」
7. 「你已經訓練完一輪判斷力。要不要試另一個題目？」
8. 「這張卡片可以稍後再填，先存草稿吧。」（給使用者退出空間，不施壓）
```

### 2.4 正確用法 vs 偷渡黑帽 對照

| ✅ 正確 | ❌ 偷渡黑帽 |
| :--- | :--- |
| 進度條顯示「3 / 9 卡片完成」 | 顯示「Lv. 3 痛點分析師」（等級） |
| 過關條件 checklist | 顯示「總分 23 / 25」（分數） |
| 「下一張卡片：___」 | 「再 6 張就升級！」（升級話術） |
| 匯出痛點身份證 | 「贏得『真痛點獵人』徽章！」（徽章） |
| 「你已完成第 5 張」 | 「擊敗 80% 的同階使用者！」（排行榜） |

## 3. 驅動力 #3 ｜ Empowerment of Creativity（賦予創造力與回饋）

### 3.1 應用範圍

這是 PainMap Worksheet 最值得驕傲的設計面向。三個體現：

1. **卡 7「自己先猜 vs AI」**（Mighty Creation Pedestal 教科書範例）
2. **卡 5「兩件事不能同時要」**（用主人翁的話寫 A/B 兩端與被犧牲的原因）
3. **自選 AI 工具**（卡 6 開放使用者自己挑 ChatGPT / Claude / Perplexity / Gemini）

### 3.2 卡 7 是 Octalysis 的經典範例 — Mighty Creation Pedestal

Octalysis 創始人 Yu-kai Chou 提出的「Mighty Creation Pedestal」：**讓使用者先創造自己的版本，再對照「正解」**。這正是 worksheet 卡 7 的設計哲學。

worksheet 第 425-427 行原文：
> 「如果你直接看 AI 回覆，會發生『**啊它說的對！**』的錯覺，這個『對』其實是 AI 順著你的問題寫的。先猜後看，你才會發現『AI 補了什麼我看不到的』『AI 漏了什麼我看到的』——**這個差異才是真的洞察**。」

實作守則：

| 元素 | 設計 |
| :--- | :--- |
| 時序鎖 | 必須先填完 4 個猜測欄位，才解鎖「請 AI 整理判斷表」按鈕 |
| 猜測欄位獨立卡片 | 4 個輸入框分別獨立，避免使用者偷看 AI 答案 |
| 對照模式 | 看 AI 回覆時，使用者的猜測仍顯示在側邊（左 / 右對照） |
| Delta 三題 | 強制要寫「最大差異 / AI 補了什麼 / 我猜對哪些 AI 沒支持」 |
| 驗證 4 檢查點 | 由使用者勾選（不是 AI 自評），讓使用者保持判斷主導權 |

### 3.3 卡 5 「兩件事不能同時要」

使用者用主人翁的話寫出 A 端、B 端、被犧牲的那一邊與原因。這是 Empowerment 的核心應用：

- 「我自己用主人翁的話寫出取捨」 = 賦予判斷權
- 「AI 協助釐清，但我可改」 = 不被 AI 答案綁架
- 「兩端拆不出來 → 中性建議回卡 3 想清楚再來」 = 賦予「拒絕」權

### 3.4 自選 AI 工具（卡 6）

worksheet 第 302-309 行給使用者選擇：

| 工具 | 適合做什麼 |
| :--- | :--- |
| ChatGPT Deep Research | 從公開資料找討論、文章 |
| Claude | 整理長文字 |
| Perplexity / Gemini DR | 補資料來源、查最新趨勢 |

設計守則：

- 不替使用者決定（站內版本仍要保留「我選哪個」的欄位）
- 提供「給小白的建議」標籤，但不強制
- `ai_evidence.ai_tool_reason` 欄位讓使用者寫「為什麼選這個」 — 強化 ownership 感

### 3.5 文案模板

```
1. 「你的判斷比 AI 的答案重要。先寫下你的猜測。」
2. 「AI 整理的判斷表只是參考。你勾選 4 個檢查點來決定它過不過關。」
3. 「兩件事的取捨怎麼寫？這是你的判斷，不是 AI 的。」
4. 「你選了哪個 AI 工具？1 句話寫下你的理由。」
5. 「猜對 / 猜錯都好。差異就是你要學的地方。」
6. 「AI 給你的回答可以反駁。請你說出『我看到 AI 沒看到的』。」
7. 「真假判斷：交給你寫，不交給 AI。」
8. 「你可以隨時編輯先前的回答。每張卡片都是你的版本。」
```

### 3.6 正確用法 vs 偷渡黑帽 對照

| ✅ 正確 | ❌ 偷渡黑帽 |
| :--- | :--- |
| 「先猜後看 AI」訓練判斷 | 「猜中即可解鎖獎勵！」（用 #6 抽獎包裝 #3） |
| 自選 AI 工具 | 「使用 X 個 AI 工具即可獲得徽章」（#3 偷渡為徽章） |
| 用主人翁的話寫取捨 | 「寫錯需重新解鎖」（#8 失去恐懼） |
| 「你的判斷比 AI 答案重要」 | 「成為超越 AI 的判斷大師」（自我膨脹） |

## 4. 驅動力 #5 ｜ Social Influence（社會影響）

### 4.1 應用範圍

PainMap Worksheet 的 #5 體現於兩處：

1. **痛點身份證可分享**（給朋友看 / 公開到部落格）
2. **Pain Atlas 集體智慧入口**（連結到 PainMap 進階版的社群痛點庫）

### 4.2 設計守則 — 避免排行榜陷阱

| ✅ 正確的 #5 | ❌ 排行榜陷阱 |
| :--- | :--- |
| 「分享你的痛點身份證」 | 「在排行榜上爬到前 100」 |
| 「Pain Atlas 已有 N 個社群痛點，你也可以貢獻」 | 「成為 Pain Atlas 的 Top Contributor」 |
| 「我也遇到」按鈕（共鳴） | 「按讚 / 評分」 |
| 「最近社群活動時間軸」 | 「熱門排行 / 最高分痛點」 |
| 匿名貢獻選項 | 「需登入用真名才能貢獻」（門檻） |

worksheet 設計上幾乎沒有 #5（因為是個人卡片填寫工具）。但站內版本可在以下地方加入 #5：

#### 4.2.1 痛點身份證 — 可分享但不排名

```
[匯出選項]
☐ Markdown（給自己看 / 部落格）
☐ JSON（備份 / 跨工具搬移）
☐ PDF（列印 / 面對面討論）
☐ 公開連結（匿名分享，不顯示作者）
```

公開連結的設計：

- URL 格式：`/painmap/atlas/{anonymous-id}`
- 顯示：判斷理由（≥100 字）、矛盾類型、痛點描述
- 不顯示：分數、評等、貢獻者排名
- 訪客行為：「我也遇到」按鈕（純共鳴，不是按讚）

#### 4.2.2 Pain Atlas 集體智慧入口

> 詳見 `painmap_brand_system.md` Pattern 4 第 344-355 行

設計守則：

- 排序選項：「最新」「最多共鳴」「你可能相關」（**沒有「最熱門」「最高分」**）
- 篩選：依產業、依矛盾類型（不依分數）
- 搜尋：以人群、場景、矛盾為主軸
- 頁面顯示：每個痛點卡片只顯示「驗證狀態」（已驗證 / 待驗證），不顯示分數

### 4.3 文案模板

```
1. 「分享你的痛點身份證，讓更多人看見這個問題。」
2. 「Pain Atlas 已有 200 個社群痛點。你也可以匿名貢獻。」
3. 「『我也遇到』— 表達共鳴，不是按讚。」
4. 「找一找 Atlas 上有沒有人遇到類似問題。」
5. 「公開連結是匿名的。沒有貢獻者排名。」
6. 「你可以參考其他人怎麼判斷類似的痛點。」
7. 「Pain Atlas 不是排行榜，是集體智慧。」
8. 「分享是選項，不是任務。」
```

### 4.4 正確用法 vs 偷渡黑帽 對照

| ✅ 正確 | ❌ 偷渡黑帽 |
| :--- | :--- |
| 「我也遇到」表達共鳴 | 「按讚數最高的痛點」（#5 偷渡 #6 排名） |
| 匿名分享降低門檻 | 「公開分享解鎖徽章」（#5 偷渡 #2 徽章） |
| 「最近社群活動」時間軸 | 「Top 10 熱門痛點」 |
| 痛點身份證可選擇分享 | 「分享後才能匯出」（強制分享） |
| Pain Atlas 隨意瀏覽 | 「貢獻 N 個才能看別人的」（互惠施壓） |

## 5. 驅動力 #4 ｜ Ownership（受限使用 — 僅限資料主權）

### 5.1 為什麼要限制？

#4 Ownership 的常見實作（IKEA 效應、收集卡牌、養成系統）容易偷渡黑帽（#2 過度成就感、#8 損失恐懼）。

PainMap Worksheet 只允許 #4 的「**資料主權**」面向：

> 「你的資料是你的。你可以匯出、刪除、編輯，無條件。」

### 5.2 具體實作 — 「資料主權」UI

#### 5.2.1 個人資料管理頁

```
┌─────────────────────────────────────────┐
│ 你的痛點卡片                             │
├─────────────────────────────────────────┤
│ □ 林老師家長 LINE         2026-04-15    │
│   [編輯] [匯出] [刪除]                   │
│                                          │
│ □ 設計師接案時間          2026-04-20    │
│   [編輯] [匯出] [刪除]                   │
└─────────────────────────────────────────┘

[匯出全部] [刪除全部]
```

#### 5.2.2 必須提供的功能

| 功能 | 規則 |
| :--- | :--- |
| 匯出 | Markdown / JSON / PDF 三種格式 |
| 編輯 | 任何時候可回去修改任何一張卡 |
| 刪除 | 單張刪除 / 全部刪除，無挽留話術 |
| 本地優先 | MVP 階段全部存 LocalStorage，無雲端帳號 |
| 透明度 | 顯示「資料存在哪裡」「誰能看見」「如何匯出」 |

#### 5.2.3 文案模板

```
1. 「你的資料存在你的瀏覽器裡。我們不上傳到雲端。」
2. 「隨時可以匯出全部資料。Markdown / JSON / PDF 三種格式。」
3. 「想刪除？一個按鈕的事。沒有挽留訊息。」
4. 「分享是選項，不是預設。」
5. 「不需要註冊帳號。打開就能用。」
6. 「沒有『會員等級』。所有功能對所有人開放。」
7. 「你的痛點卡片只屬於你。」
8. 「我們不收集分析數據。只有你看得到自己填什麼。」
```

### 5.3 ❌ 禁止的 #4 偽裝

| 禁止 | 原因 |
| :--- | :--- |
| 「集卡牌：填完 9 張卡片解鎖痛點獵人徽章」 | IKEA 效應偷渡 #2 + 收集成癮 |
| 「完成 5 個痛點身份證可解鎖進階模板」 | 解鎖機制 = 偷渡 #6 稀缺 |

| 「一鍵刪除前需確認 3 次 + 最後挽留訊息」 | 偽裝 #4 實則 #8 |

## 6. 跨驅動力的綜合守則

### 6.1 每個 UI 元素的審查清單

新增任何介面元素時，逐項問：

- [ ] 它鼓勵的是「下一步行動」還是「持續黏著」？（前者好，後者疑慮）
- [ ] 它觸發的是「成就感」還是「失去恐懼」？（前者好，後者禁止）
- [ ] 它依賴「使用者的判斷」還是「系統的評分」？（前者好，後者禁止）
- [ ] 它鼓勵「離開後仍可獨立練習」還是「必須留在站上」？（前者好，後者疑慮）
- [ ] 移除這個元素，整體流程還能完整嗎？（如果還行，思考是否該移除）

### 6.2 文案 / 視覺 / 互動三層審查

| 層級 | 檢查 |
| :--- | :--- |
| 文案 | 不出現「點子 / idea / 評分 / 等級 / 排名 / 徽章 / 連續 / 過期 / streak / FOMO」 |
| 視覺 | 不使用大面積紅色（焦慮）、星等圖示、徽章圖示、分數數字 |
| 互動 | 不設倒數計時器、不設強制連續登入、不設過期失效機制 |
