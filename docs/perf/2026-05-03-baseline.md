## Landing first-load (gzipped)

| Asset | Before | After | Δ |
| --- | --- | --- | --- |
| `index-*.js` (entry chunk) | 129,856 B | **70,404 B** | **−45.8 %** |
| `react-vendor-*.js` (new) | — | 60,437 B | +60,437 B (cacheable) |

| **First-paint critical (entry+vendor+css)** | **149,191 B** | **150,267 B** | +1,076 B (≈0.7 %) |

**取捨**：第一次造訪幾乎打平（多 1 KB gz）；回訪 / 後續部署只動業務 chunk → vendor 走 304，淨省約 60 KB gz。對使用者旅程「來一次寫完 9 卡」是 net-positive。

## Font payload (same-origin self-host)

| File | Size | Source |
| --- | --- | --- |
| `geist-sans-variable.woff2` | 68 KB | npm `geist@1.5.1` → `public/fonts/` |
| `geist-mono-variable.woff2` | 70 KB | npm `geist@1.5.1` → `public/fonts/` |

Variable woff2 + `font-display: swap` + `<link rel="preload">` → no perceptible FOUT on a warm cache; cold cache shows fallback fonts for ≤100 ms then swaps. Previously: 2 jsdelivr stylesheet round-trips before any font byte arrived.

## Vendor chunk policy (`vite.config.ts`)

```
react-vendor    → react, react-dom, scheduler
tanstack-vendor → @tanstack/*  (lazy — only worksheet routes need it)
radix-vendor    → @radix-ui/*  (lazy — only worksheet card forms need it)
icons-vendor    → lucide-react (lazy — only routes that import icons)
```

Landing entry now `import`s only `react-vendor` (verified: 1 sibling chunk reference). Other vendor chunks are loaded lazily by route-level dynamic imports.

## Bundle visualizer

`dist/stats.html` (~1 MB) emitted by `rollup-plugin-visualizer` on every `npm run build`. Reflects the **server (Cloudflare Worker) bundle** — important for staying under Cloudflare Worker size limits. For client-side analysis, use `source-map-explorer dist/client/assets/*.js` ad-hoc.

## What we did NOT do (and why)

| Skipped | Reason |
| --- | --- |
| SSR streaming refactor | TanStack Start defaults already stream |
| Image optimization | Landing has no large images (Hero is pure CSS) |
| Critical-CSS extraction | 19 KB gz CSS already small enough; tooling cost > benefit |
| Tailwind class purge audit | Tailwind v4 source-aware mode already minimises emit |
| Self-host Noto Sans TC | Google Fonts already serves CJK with `unicode-range` subsetting (browser only fetches the slice it needs, ~50 KB per slice). Self-hosting 600 unique glyphs as one woff2 ≈ 60-80 KB — same total payload, plus ongoing "re-subset every time copy adds a new character" maintenance. Net negative. |
| Landing static prerender | Tried `tanstackStart.prerender` for `/`. Crashed at build time: the bundled Cloudflare preset emits `worker-entry-*.js`, but the prerender plugin self-fetches via a Node-style `dist/server/server.js` entry that doesn't exist. Right answer is to enable Cloudflare HTML caching for `/` at the worker layer (not a vite-time change) — out of source-code scope. |

## Round 2 follow-ups (2026-05-03)

After the initial round shipped, three additional changes landed in the
same session under the same baseline doc to keep the perf history
together:

### `learn.worksheet.07` chunk

| | Before | After | Δ |
| --- | --- | --- | --- |
| Raw | 183,087 B | 49,365 B | −73.0 % |
| **Gzipped** | **56,495 B** | **17,688 B** | **−68.7 %** |

Achieved by replacing `react-markdown` + `remark-gfm` (+ transitive
`micromark`) with `marked` + `DOMPurify` in `MarkdownView.tsx`.
`marked` was already shipped (Card 10 PDF export uses it dynamically),
DOMPurify is a jsPDF transitive — both free at the dependency level.

### Card 8 interview-question judge

Expanded `card8.no_selling_questions` LLM prompt from "selling-mode
only" to a 4-category taxonomy:
- A. **Sells** a solution to the interviewee
- B. **Hypothetical** ("if X existed, would you use it?")
- C. **Leading** ("isn't X frustrating?")
- D. **Binary framing** ("A or B?")

Adds bad/why/rewrite tables. Same JudgeKind, same wiring — pure prompt
quality bump. Catches more bad-interview-question patterns before
users go bother real interviewees.

### Card 9 verdict audit (rule-bending narrow exception)

Card 9's iron rule "AI is forbidden from the verdict layer" preserved.
Added a *user-initiated, supplementary* alignment check: a low-key
outline button "請 AI 紅隊檢查（參考用）" that opens a Dialog with an
LLM-generated verdict on whether the 100-word reason aligns with
evidence in cards 1-8. Result is read-only — does NOT write to verdict
fields, does NOT block ExitGate. Entirely opt-in.

New `JudgeKind`: `card9.verdict_audit`.
New component: `src/components/worksheet/card09/VerdictAuditDialog.tsx`.
