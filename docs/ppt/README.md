# PainMap Worksheet — 介紹投影片（5 頁）

> **用途**：產品介紹用簡報，投影片只講概念與大架構，**細節由現場 demo 補完**
> **對象**：沒有程式基礎、不熟 AI 的一般人
> **總時長**：投影片約 3–4 分鐘 + 現場 Demo 5–10 分鐘
> **設計原則**：每頁 3 秒讓觀眾抓到核心，每頁一個概念、一句大字、一個視覺

## 設計鐵律

1. **3 秒原則** — 觀眾掃一眼就要懂
2. **一頁一概念** — 不堆訊息、不擠細節
3. **大字優先** — 主訊息字級至少 40pt
4. **不講技術** — 沒有程式語言、框架、儲存名詞
5. **不講細節** — 流程、案例、教學全部留給 Demo
6. **講感受不講功能** — 「你會放心」優先於「資料儲存在本機」

## 核心理論骨架：SECI 知識螺旋

整份簡報背後的設計理念，源自 **野中郁次郎 & 竹內弘高（1995）SECI 模型**。
這是一個說明「個人隱性知識如何透過社會互動變成組織顯性知識，再回到個人能力」的螺旋上升模型。

| SECI 階段 | 知識轉化 | 大白話 | 在工具裡的對應 |
| :--- | :--- | :--- | :--- |
| **Socialization 社會化** | 隱性 → 隱性 | **聽見** | 你已經聽到那句抱怨 |
| **Externalization 外化** | 隱性 → 顯性 | **寫下** | 卡 1-5：把感覺攤開成字 |
| **Combination 組合** | 顯性 → 顯性 | **對照** | 卡 6-7：你的看法 vs AI 證據 |
| **Internalization 內化** | 顯性 → 隱性 | **行動** | 卡 8-9：訪談 + 下決定 + 去做 |

**為什麼選 SECI？**

PainMap Worksheet 的真正價值不在「工具本身」，而在工具承擔了 SECI 螺旋裡最關鍵、也最缺乏支援的「**外化（Externalization）**」階段——把腦中說不清楚的隱性感覺，變成可以看、可以分享、可以再驗證的顯性字句。

走完一圈後，使用者的判斷力（內化能力）會增強，下次聽到類似抱怨時更敏銳，**形成螺旋上升**。

**呈現原則**：

- **02 大架構頁** 用螺旋圖完整呈現 4 階段（核心）
- 其他頁用大白話「聽見 / 寫下 / 對照 / 行動」即可
- SECI 原文標籤一律用小字、淺色，不喧賓奪主
- 對小白觀眾不需要主動解釋 SECI 名稱，**懂的人會會心一笑、不懂的人也看得懂**
